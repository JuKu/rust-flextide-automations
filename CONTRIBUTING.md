# Contributing to Flextide

## Thank You
Thank you for your interest in contributing to Flextide! This project aims to provide a high‑quality, secure, and powerful workflow automation platform.

## How to Contribute

### 1. Fork the repository
Create your own fork and clone it locally.

### 2. Create a feature branch
Use conventional commit naming.

### 3. Rust Guidelines
- Follow clippy lint suggestions
- Use sqlx with compile‑time validation
- Write unit tests for core logic

### 4. UI Guidelines
- Use TypeScript
- Follow project Tailwind config and component patterns
- Keep editor interactions smooth and optimized

### 5. Submitting Pull Requests
- PR must describe the change clearly
- Add tests where needed
- Keep commits clean

## Code of Conduct
Respect everyone. No harassment, no discrimination, no toxicity.

## Getting Help
Open an issue or start a discussion on GitHub.
